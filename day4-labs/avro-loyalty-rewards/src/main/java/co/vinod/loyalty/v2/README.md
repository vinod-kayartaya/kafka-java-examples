# V2 User Guide - Avro with Kafka and Schema Registry

## Objective

- Creating Kafka producers using Avro
- Creating Kafka consumers using Avro
- Using Schema Registry
- Automatic schema registration
- Publishing strongly-typed events to Kafka
- Consuming Avro events from Kafka

# Learning Outcomes

After completing this exercise, you should be able to:

- Explain why Avro is commonly used with Kafka
- Produce Avro messages to Kafka topics
- Consume Avro messages from Kafka topics
- Configure KafkaAvroSerializer
- Configure KafkaAvroDeserializer
- Understand how Schema Registry works
- Explain how schemas are automatically registered

# Business Scenario

An e-commerce company operates a loyalty rewards system.

Whenever a customer makes a purchase, a purchase event is generated.

The Purchase Service publishes the event to Kafka.

The Rewards Service consumes the event and awards loyalty points.

# Architecture

```text
Purchase Service
       |
       | CustomerPurchase Event
       |
       v
Kafka Producer
       |
       v
Kafka Topic
(customer-purchases)
       |
       v
Kafka Consumer
       |
       v
Rewards Service
```

# Project Components

## Avro Schema

Location:

```text
schemas/v2/CustomerPurchase.avsc
```

Schema:

```json
{
  "type": "record",
  "name": "CustomerPurchase",
  "namespace": "co.vinod.loyalty.avro.v2",

  "fields": [
    {
      "name": "customerId",
      "type": "string"
    },
    {
      "name": "purchaseAmount",
      "type": "double"
    },
    {
      "name": "pointsEarned",
      "type": "int"
    }
  ]
}
```

## Generated Java Class

Generated by Maven:

```text
co.vinod.loyalty.avro.v2.CustomerPurchase
```

This class represents the event contract.

## PurchaseProducer

Location:

```text
co.vinod.loyalty.v2.PurchaseProducer
```

Responsibilities:

- Create CustomerPurchase object
- Serialize using Avro
- Register schema automatically
- Publish event to Kafka

## PurchaseConsumer

Location:

```text
co.vinod.loyalty.v2.PurchaseConsumer
```

Responsibilities:

- Subscribe to Kafka topic
- Retrieve schema from Schema Registry
- Deserialize Avro message
- Display purchase details

# Infrastructure Components

## Kafka Cluster

Responsible for:

- Receiving messages
- Storing messages
- Delivering messages

## Schema Registry

Responsible for:

- Storing schemas
- Versioning schemas
- Providing schemas to producers and consumers

# Message Flow

## Step 1

Producer creates event

```java
CustomerPurchase purchase =
        CustomerPurchase.newBuilder()
                .setCustomerId("C1001")
                .setPurchaseAmount(1500)
                .setPointsEarned(150)
                .build();
```

## Step 2

KafkaAvroSerializer extracts schema

```text
CustomerPurchase
       ↓
Schema
```

## Step 3

Schema Registry registers schema

```text
customer-purchases-value
```

## Step 4

Schema Registry returns Schema ID

Example:

```text
Schema ID = 1
```

## Step 5

Producer sends message

```text
Kafka Topic
customer-purchases
```

## Step 6

Consumer receives message

## Step 7

Consumer retrieves schema

```text
Schema ID
     ↓
Schema Registry
     ↓
Schema
```

## Step 8

Consumer reconstructs object

```java
CustomerPurchase
```

# Configuration File

Location:

```text
src/main/resources/v2/application.properties
```

Example:

```properties
bootstrap.servers=localhost:9092

topic.name=customer-purchases

producer.key.serializer=org.apache.kafka.common.serialization.StringSerializer
producer.value.serializer=io.confluent.kafka.serializers.KafkaAvroSerializer

consumer.group.id=rewards-group

consumer.key.deserializer=org.apache.kafka.common.serialization.StringDeserializer
consumer.value.deserializer=io.confluent.kafka.serializers.KafkaAvroDeserializer

consumer.auto.offset.reset=earliest

schema.registry.url=http://localhost:8081

specific.avro.reader=true
```

# Key Concepts Learned

## KafkaAvroSerializer

Converts Java objects into Avro messages.

Used by:

```java
PurchaseProducer
```

## KafkaAvroDeserializer

Converts Avro messages back into Java objects.

Used by:

```java
PurchaseConsumer
```

## Schema Registry

Stores and manages schemas centrally.

Benefits:

- Version control
- Schema validation
- Compatibility checking

## Subject

A named schema stored in Schema Registry.

Example:

```text
customer-purchases-value
```

## Schema ID

Each schema receives a unique identifier.

Example:

```text
1
```

The Schema ID is embedded inside Kafka messages.
